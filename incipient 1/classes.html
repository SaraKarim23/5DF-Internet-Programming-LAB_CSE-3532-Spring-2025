Classes of C++

Introduction to Object-Oriented Programming (OOP):

Object-Oriented Programming, or OOP, is a way of writing programs by focusing on objects—real-world or logical entities that have properties (data) and behaviors (functions). Instead of just writing procedures and functions, OOP allows us to model things more naturally, just like we see in daily life.

In C++, OOP helps break down complex problems into smaller, manageable parts by organizing code into classes and objects. This approach makes code easier to understand, modify, reuse, and scale.

Object-oriented programming has several advantages over procedural programming:

  • OOP is faster and easier to execute
  • OOP provides a clear structure for the programs
  • OOP makes it possible to create full reusable applications with less code and shorter development time
  • OOP helps to keep the C++ code DRY "Don't Repeat Yourself", and makes the code easier to maintain, modify and debug


    • C++ Classes and Objects

    In C++, classes and objects are the heart of Object-Oriented Programming (OOP). They allow you to design your code in a way that mirrors how things work in the real world—by grouping data and behavior into single units.

    What is a Class?

    A class is like a blueprint or a template. It defines what an object should look like and how it should behave. Inside a class, you can declare variables (called data members) and functions (called member functions) that work with those variables.

    Example -
    class Car {
    public:
    string brand;
    int speed;

    void drive() {
    cout << "The car is moving." << endl;
    }
    };

    This defines a Car class that has two properties (brand, speed) and a behavior (drive()).

    What is an Object?

    An object is an actual instance of a class. If the class is a design, the object is the real thing built from it.

    Example -
    Car myCar; // Object created from the class Car
    myCar.brand = "BMW";
    myCar.speed = 120;
    myCar.drive();
    Why Use Classes and Objects?
  • Keep code clean, organized, and reusable
  • Easy to model real-life systems (like a Student, Book, BankAccount, etc.)
  • Essential for building large software in teams or modular systems

    • Class Methods

    In C++, a class method (also called a member function) is a function that belongs to the class.

    There are two ways to define functions that belongs to a class:

  • Inside class definition
  • Outside class definition

    • Method Define Inside a Class
    class Person {
    public:
    void sayHello() {
    cout << "Hello!" << endl;
    }
    };

    Here, sayHello() is a method that belongs to the Person class.

    Using the Method with an Object
    Person p;
    p.sayHello(); // Calling the method

    The method is called using the object followed by a dot (.) and the method name.

    Method Defined Outside the Class

    You can also write the method outside the class using the scope resolution operator :: -

    class Box {
    public:
    void show();
    };

    void Box::show() {
    cout << "This is a box." << endl;
    }

    Why Class Methods Matter?
  • They make code modular and reusable
  • Allow each object to act on its own data
  • Keep related logic inside the class for better readability and structure


    • Constructor in C++

    A constructor is a special function in C++ that runs automatically when you create an object from a class. Its main job is to initialize object data—so you don’t have to set every value manually after making an object.

    Key Characteristics of a Constructor:
  • The constructor has the same name as the class
  • It doesn’t have a return type (not even void)
  • It runs automatically when an object is created

    • Why Constructors Are Useful:
  • Automatically set up objects with default or specific values
  • Make object creation cleaner and more efficient
  • Help prevent uninitialized or inconsistent object states

    • Basic Example:
    class Student {
    public:
    string name;
    int age;

    // Constructor
    Student() {
    name = "Unknown";
    age = 0;
    }
    };

    int main() {
    Student s1;
    cout << s1.name << " " << s1.age;
    // Output: Unknown 0
    }

    Here, as soon as we create s1, the constructor sets default values.

    Parameterized Constructor
    class Student {
    public:
    string name;
    int age;

    Student(string n, int a) {
    name = n;
    age = a;
    }
    };

    Student s2("Aisha", 20);

    Now the object is created and initialized in a single line.

    Multiple Constructors (Constructor Overloading)
    Student() { ... }
    Student(string n) { ... }
    Student(string n, int a) { ... }


    Encapsulation in C++

    Encapsulation is one of the core principles of Object-Oriented Programming in C++. It means wrapping data and related functions together in a single unit (usually a class), and restricting direct access to some of the object's components. Think of it like putting important things in a box and only allowing access through specific doors (functions).

    In Technical Terms:
    Encapsulation is achieved using access specifiers:
  • private → hidden from outside
  • public → accessible from outside
  • protected → used in inheritance

    • Example:
    class BankAccount {
    private:
    double balance;

    public:
    void setBalance(double amount) {
    if (amount >= 0) balance = amount;
    }

    double getBalance() {
    return balance;
    }
    };

    Here:

  • balance is private, so it can’t be accessed directly.
  • Instead, you use setBalance() and getBalance() to interact with it—safe and controlled.



    • Inheritance in C++

    Inheritance is a key feature of Object-Oriented Programming in C++ that allows one class to inherit properties and behaviors from another class. This means we can reuse existing code, extend it, or customize it—without starting from scratch.

    Basic Idea:
  • The class that gives its features is called the base class (or parent class)
  • The class that receives those features is called the derived class (or child class)

    • Syntax Example:
    // Base class
    class Vehicle {
    public:
    string brand = "Ford";
    void honk() {
    cout << "Tuut, tuut! \n" ;
    }
    };

    // Derived class
    Car: public Vehicle {
    public:
    string model = "Mustang";
    };

    int main() {
    Car myCar;
    myCar.honk();
    cout << myCar.brand + " " + myCar.model;
    return 0;
    }
    Types of Inheritance in C++:
    1.Single Inheritance – One base class, one derived class
    2.Multilevel Inheritance – A class derived from a derived class
    3.Multiple Inheritance – A class inherits from more than one base class
    4.Hierarchical Inheritance – Multiple derived classes from one base class
    5.Hybrid Inheritance – A mix of the above types

    C++ Polymorphism

    Polymorphism is a core concept in C++ that allows the same function or operator to behave differently depending on the context. The word itself means “many forms.” In programming, this means we can use a single name to represent different functionalities.

    With polymorphism, you can write cleaner, more flexible code that adapts to different situations—without rewriting everything.

    Types of Polymorphism in C++:

    1. Compile-Time Polymorphism (Static): The decision about which function to call is made at compile time.

    Example: Function Overloading

    class Math {
    public:
    int add(int a, int b) {
    return a + b;
    }

    double add(double a, double b) {
    return a + b;
    }
    };

    Here, the function add() works with both integers and doubles.


    2. Run-Time Polymorphism (Dynamic): The decision of which method to call is made while the program is running.

    Example: Virtual Function

    class Animal {
    public:
    virtual void sound() {
    cout << "Some sound" << endl;
    }
    };

    class Dog : public Animal {
    public:
    void sound() override {
    cout << "Bark" << endl;
    }
    };

    void makeSound(Animal* a) {
    a->sound();
    }
    Dog d;
    makeSound(&d);
    // Output: Bark (not "Some sound")


    Exceptions in C++

    An exception is an unexpected event or error that occurs during program execution. Instead of stopping the program, C++ lets you “throw” an error and “catch” it somewhere safe.

    Basic Syntax:
    try {
    // Code that might throw an error
    } catch (type variable) {
    // Code to handle the error
    }
    Simple Example:
    #include <iostream>
    using namespace std;

    int main() {
    try {
    int a = 5, b = 0;
    if (b == 0)
    throw "Cannot divide by zero!";
    cout << a / b;
    } catch (const char* msg) {
    cout << "Error: " << msg << endl;
    }
    return 0;
    }

    Output:
    Error: Cannot divide by zero!

    Keywords:
  • try – Wraps code that might cause an exception
  • throw – Sends the exception (error) out of the block
  • catch – Receives and handles the exception

    • Why Use Exceptions?
  • Makes your code more reliable and readable
  • Keeps programs from crashing unexpectedly
  • Helps handle runtime errors cleanly
  • Useful in real applications: file handling, memory errors, user inputs, etc.
